The destination. Each of 4 wheels has **independent steering and independent driving**.
The combination produces the most capable kinematic in wheeled robotics.

Swerve is what you'll find on:

- Modern FRC (FIRST Robotics Competition) winning robots
- The Boston Dynamics Spot's wheel-leg hybrid (in spirit)
- High-end logistics robots in warehouses
- Most agile-mobility research platforms

## The single change from Phase 4

Phase 4: front wheels independently steered, rear wheels straight.
Phase 5: **rear wheels also independently steered.**

That's it. The added degree of freedom unlocks:

- Sideways motion (Phase 2's capability)
- In-place rotation (Phase 1's capability)
- Field-oriented driving (NEW — drive a direction independent of heading)
- Simultaneous translate-and-rotate-and-strafe in any combination

In effect, swerve = mecanum's freedom + Ackermann's precision.

## The kinematics

For each module at body-frame position `(l_x, l_y)`:

```
Velocity vector at the module:
  v_i_x = v_b_x - ω_b_z · l_y
  v_i_y = v_b_y + ω_b_z · l_x

Module commands:
  v_i   = √(v_i_x² + v_i_y²)            ← wheel speed (scalar)
  φ_i   = arctan2(v_i_y, v_i_x)         ← steering angle
```

Notice the input is *3D* (`v_x, v_y, ω_z`) — same as mecanum. The kinematics maps that to
**4 × (speed + angle) = 8 module outputs**.

It's a fully determined system (3 inputs → 8 outputs), so 8 wheel commands can produce any
chassis motion. The redundancy means swerve naturally has some optimization room (e.g.
minimize angular changes to extend mechanical lifetime).

## The capability demo

I encoded a 6-step sequence that exhibits every capability swerve adds:

```
1. STRAIGHT           v_x=0.3,  v_y=0,    ω=0       (Phase 1 baseline)
2. SIDEWAYS           v_x=0,    v_y=0.3,  ω=0       (Phase 2 capability)
3. DIAGONAL           v_x=0.3,  v_y=0.3,  ω=0       (Phase 2 capability)
4. SIDEWAYS + ROTATE  v_x=0,    v_y=0.3,  ω=0.3     (NEW — only swerve does this cleanly)
5. IN-PLACE ROTATION  v_x=0,    v_y=0,    ω=0.5     (NEW after Phase 3,4)
6. ALL 3 AXES         v_x=0.3,  v_y=0.2,  ω=0.2     (Pure swerve)
```

Each step prints the per-module steering angle and speed. Example log for in-place rotation:

```
>>> 5. IN-PLACE ROTATION (제자리) for 3s <<<
--- cmd_vel: v_x=+0.00, v_y=+0.00, ω_z=+0.50 ---
  FL: speed=0.165 m/s, steering=+135.0°
  FR: speed=0.165 m/s, steering= +45.0°
  BL: speed=0.165 m/s, steering=-135.0°
  BR: speed=0.165 m/s, steering= -45.0°
```

All four wheels same speed, four different angles tangent to the rotation circle. That's
the swerve signature.

## Code structure

```python
class SwerveKinematics(Node):
    def __init__(self):
        super().__init__('swerve_kinematics')
        self.modules = {
            'FL': (+0.240, +0.225),
            'FR': (+0.240, -0.225),
            'BL': (-0.240, +0.225),
            'BR': (-0.240, -0.225),
        }
        # ... publisher, subscriber, sequence state machine ...

    def calculate_swerve(self, v_x, v_y, w_z):
        results = {}
        for name, (lx, ly) in self.modules.items():
            v_ix = v_x - w_z * ly
            v_iy = v_y + w_z * lx
            v_i = math.sqrt(v_ix**2 + v_iy**2)
            phi_i = math.atan2(v_iy, v_ix) if v_i > 1e-6 else 0.0
            results[name] = (v_i, phi_i)
        return results
```

Per-module independence is the architectural point. There's no central "left vs right"
asymmetry — every module computes its own answer based on its own location.

## What swerve cannot do (the honest list)

Swerve still has limits:

1. **Mechanical complexity.** Each module is a small robot in itself — steering motor,
   drive motor, encoders, slip-ring (for unlimited rotation). 8 motors total, vs 4 for
   mecanum and 2 for diff.

2. **Cost.** A research-grade swerve module costs $1k-3k each. A swerve robot is easily
   $20k+ in mechanical hardware alone.

3. **Control complexity.** Module steering must reach the target angle *before* drive
   speed is applied, or you'll get a brief unwanted velocity vector. Real swerve
   controllers do **module velocity vector smoothing**.

4. **Power draw.** 8 active motors means 2× the idle power of differential.

5. **Calibration.** Misalignment between modules → curved "straight" motion. Regular
   calibration is required.

So swerve isn't a strict improvement; it's the right choice when **agility matters more
than cost or simplicity**.
